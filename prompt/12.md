# 1. 系統總覽（Data Pipeline）

人類圖計算本質是「天文座標分層切割系統」：

```txt
出生時間 + 地點
↓
天文計算（行星黃經 longitude）
↓
Human Design Mandala 對應
↓
Gate（64分割）
↓
Line（6分割）
↓
Color（6分割）
↓
Tone（6分割）
↓
Base（5分割）

核心概念：
360° 黃道被逐層切割為：
64 → 6 → 6 → 6 → 5

5. Gate → Line → Color → Tone → Base
5.1 核心切割比例
Gate  = 360 / 64 = 5.625°
Line  = Gate / 6
Color = Line / 6
Tone  = Color / 6
Base  = Tone / 5
5.2 計算邏輯
輸入：
longitude: number
gateStartLongitude: number
5.3 完整計算
const GATE_SIZE = 360 / 64
const LINE_SIZE = GATE_SIZE / 6
const COLOR_SIZE = LINE_SIZE / 6
const TONE_SIZE = COLOR_SIZE / 6
const BASE_SIZE = TONE_SIZE / 5

export function calculateHD(
  longitude: number,
  gateStartLongitude: number
) {
  const offset =
    longitude - gateStartLongitude

  const gate =
    Math.floor(offset / GATE_SIZE) + 1

  const gateOffset =
    offset % GATE_SIZE

  const line =
    Math.floor(gateOffset / LINE_SIZE) + 1

  const lineOffset =
    gateOffset % LINE_SIZE

  const color =
    Math.floor(lineOffset / COLOR_SIZE) + 1

  const colorOffset =
    lineOffset % COLOR_SIZE

  const tone =
    Math.floor(colorOffset / TONE_SIZE) + 1

  const toneOffset =
    colorOffset % TONE_SIZE

  const base =
    Math.floor(toneOffset / BASE_SIZE) + 1

  return {
    gate,
    line,
    color,
    tone,
    base,
  }
}

7. Color / Tone / Base（結構來源）
7.1 定義

Tone  = Color 再細分

7.2 工程化切割模型
360°
→ 64 Gates
→ 6 Lines
→ 6 Colors
→ 6 Tones
→ 5 Bases
7.3 注意事項

再進行 offset 計算
不能直接從出生時間推導
